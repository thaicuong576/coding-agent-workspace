## 1. Objective: Resolve Subtitle Bunching & Readability Issues
The user reported that translated Vietnamese subtitle blocks were "bunched up" (multiple sentences grouped into long, unreadable blocks lasting 8-11 seconds). 

## 2. Diagnosis
*   By default, when Whisper transcribes continuous speech with few pauses, `stable-ts` groups segments into very long cues.
*   Because the translator translates line-by-line using the source SRT structure, the resulting Vietnamese text inherits these massive cues, wrapping into 4-6 lines and crowding the video frame.

## 3. Solution Implemented
In `pipeline/subtitles.py` (inside the `transcribe_audio` function), we integrated `stable-ts`'s built-in splitting controls to force shorter, more readable subtitle blocks:
```python
    # Split segments by gaps (pauses) and length constraints to avoid extremely long cues
    result.split_by_gap(0.5)
    if detected_lang in ["zh", "ja", "ko"]:
        result.split_by_length(max_chars=22)
    else:
        result.split_by_length(max_chars=40, max_words=12)
```
*   **Gap Split**: Splits segments if there is a silence/gap of at least `0.5` seconds.
*   **Length Split**:
    *   For Chinese/Japanese/Korean (CJK): Splits if a segment exceeds `22` characters (since double-byte characters are wide).
    *   For other languages (e.g. English, Vietnamese): Splits if a segment exceeds `40` characters or `12` words.
