## Active Task: Double-Subtitle scanning & masking optimization
We audited the secondary zone (double-subtitles) scanning logic in `pipeline/video.py` and identified two major flaws causing false-positive mask bands around slide text or code snippets:
1.  **Vertical Vote Stacking (No Frame Grouping)**: A single frame with multiple blocks of text (like a code block or multi-line paragraph) adds multiple votes to the same Y coordinates, inflating the density and triggering false double-subtitle zones.
2.  **Lack of Temporal Frame-Correlation**: Candidates are scanned on all frames rather than only on frames where a primary speech subtitle is actively visible.

### Proposed Solution:
*   Limit secondary candidates to frame indices that contain a primary subtitle (`matches_srt=True`).
*   Group candidate blocks by `frame_idx` and use a binary mask per frame so each frame contributes at most 1 vote to a coordinate, eliminating vertical stacking inflation.
*   Maintain the 50% threshold of the primary peak to detect true bilingual subtitles.
