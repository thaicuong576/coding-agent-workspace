## 1. Objective: Pacing and Punctuation-Aware Subtitle Splitting
Optimized the translation system prompt and the splitting pipeline to:
1. Shorten translated subtitle blocks to prevent Siren TTS from speaking too fast.
2. Keep numbers as digits (e.g., `RTX 4090`, `4000`, `96%`) since the TTS can now read digits natively.
3. Prevent compound words (e.g., "thông minh", "tất cả") from being broken across separate timestamps.
4. Support English as a target language option (with specific English TTS model & voice).

## 2. Changes Implemented
*   **System Prompt Conciseness**:
    *   Updated the system prompt in `pipeline/subtitles.py` (`translate_srt`) to enforce conciseness (lược dịch), stripping out fluff/filler words (like "thì", "mà", "là", "thực sự", "hoàn toàn").
    *   Instructed the LLM to preserve numbers and symbols as digits instead of spelling them out in words.
    *   Made the system prompt adapt dynamically based on `target_language` (supporting custom rules for Vietnamese and English).
*   **CLI Argument `--target-lang`**:
    *   Exposed `--target-lang` in `repurpose.py` to allow dynamic target language configuration (defaults to `vi`).
*   **Punctuation-Aware Splitter**:
    *   Upgraded `wrap_srt_text` in `pipeline/subtitles.py`.
    *   Instead of dividing by word count indexes, it scans a sliding window from right to left, prioritizing splits immediately after a punctuation mark (`,`, `.`, `?`, `!`, `;`, `:`).
    *   As a fallback, it splits around major conjunctions (e.g., "và", "nhưng", "hoặc").
    *   Increased the maximum words threshold to `14` to prevent over-segmentation.
